02Three-Tier Architecture

Each tier has a distinct responsibility. Communication flows through HTTP callbacks and direct DB access — no SSH tunnels or file polling.

Tier 1 — Claude Cloud

Anthropic infrastructure for intelligence work

  • Remote Task Backtesting pipeline (on-demand)
  • Scheduled Pre-market analysis (daily 7am)
  • Scheduled Strategy review (weekly)
  • Future Agent team coordination

Tier 2 — Phoenix Server

Coolify VPS running all core services

  • FastAPI REST API, auth, callbacks
  • PostgreSQL All state: agents, trades, logs
  • asyncio Task Runner (local backtesting)
  • Docker Trading worker containers
  • React Dashboard + monitoring

Tier 3 — Communication

How components talk to each other

  • HTTP POST Cloud → API callbacks
  • REST Dashboard → API polling
  • SQLAlchemy Worker → DB direct
  • Docker SDK API → Worker control

03Agent Lifecycle State Machine

Every agent progresses through a gated lifecycle — from creation through backtesting, approval, and finally live trading. Each transition is an explicit API call.

Agent Status Transitions

%%{init: {'theme': 'dark', 'themeVariables': {'primaryColor': '#6c5ce7', 'primaryTextColor': '#e8e8f0', 'lineColor': '#4a4f6a', 'secondaryColor': '#1c2039'}}}%%
stateDiagram-v2
    [*] --> CREATED: Dashboard Wizard
    CREATED --> BACKTESTING: run_backtest()
    BACKTESTING --> BACKTEST_COMPLETE: Pipeline succeeds
    BACKTESTING --> CREATED: Pipeline fails (retry)
    BACKTEST_COMPLETE --> PAPER: User approves (paper)
    BACKTEST_COMPLETE --> APPROVED: User approves (live)
    PAPER --> RUNNING: User promotes
    APPROVED --> RUNNING: User promotes
    RUNNING --> PAUSED: User pauses
    PAUSED --> RUNNING: User resumes
    RUNNING --> [*]: User deletes

Worker Status (Docker Container)

%%{init: {'theme': 'dark', 'themeVariables': {'primaryColor': '#00cec9', 'primaryTextColor': '#e8e8f0', 'lineColor': '#4a4f6a', 'secondaryColor': '#1c2039'}}}%%
stateDiagram-v2
    [*] --> STOPPED: Agent created
    STOPPED --> STARTING: promote triggers
    STARTING --> RUNNING: Container healthy
    STARTING --> ERROR: Container crash
    RUNNING --> STOPPED: Pause / Delete
    ERROR --> STARTING: Retry

Agent Status Meanings

  • CREATED — row exists, no backtest yet (or failed retry)
  • BACKTESTING — pipeline actively running
  • BACKTEST_COMPLETE — results ready for review
  • PAPER / APPROVED — user approved; awaiting promotion
  • RUNNING — live Docker worker active
  • PAUSED — worker stopped, can resume

Key API Transitions

  • POST /agents — Creates agent + starts backtest
  • POST /agents/:id/approve — Requires BACKTEST_COMPLETE
  • POST /agents/:id/promote — Starts Docker worker
  • POST /agents/:id/command — pause / resume / switch_mode
  • DELETE /agents/:id — Stops worker + removes agent

Manifest Sections

  • identity — name, channel, analyst, character type
  • modes — conservative / moderate / aggressive thresholds
  • risk — max position %, daily loss %, concurrent positions
  • models — primary model type, accuracy, AUC-ROC, version
  • rules — discovered patterns as trading rules
  • knowledge — top features, analyst profile, channel summary
  • tools — enabled Python scripts for the live agent
  • skills — Markdown skill files for agent behavior
  • credentials — Discord token, Robinhood, Phoenix API key

Character Types (Auto-Detected)

  • aggressive-scalper — high frequency, tight stops
  • balanced-intraday — default, moderate risk (fallback)
  • conservative-swing — longer holds, wider stops
  • momentum-rider — trend following, trailing stops
  • contrarian-value — mean reversion, oversold entries
  • Character determines mode overrides for risk thresholds
